# Signal Lab — Rubric

Общий балл: **100**. Порог сильного выполнения: **75+**.

---

## 1. Рабочее приложение и стек — 25 баллов

| Баллы | Что видит интервьюер |
|-------|---------------------|
| 0–8 | Приложение не стартует или стек соблюдён частично. Docker Compose падает, нет health endpoint, frontend пустой. |
| 9–16 | Приложение стартует. Frontend + backend + PG работают. Есть базовый UI и API. Часть стека может быть подключена формально (RHF/TanStack без реального использования). |
| 17–21 | Весь стек используется по назначению. Форма через RHF, запросы через TanStack Query, shadcn компоненты, NestJS с Prisma. Структура чистая. |
| 22–25 | Всё из 17–21, плюс: hot reload работает, .env.example полный, Swagger на месте, seed опционален но полезен. Код читается легко. |

---

## 2. Observability — 25 баллов

| Баллы | Что видит интервьюер |
|-------|---------------------|
| 0–8 | Observability заявлена, но verification walkthrough не проходится. Dashboard пустой, Loki не подключен, Sentry DSN placeholder. |
| 9–16 | Часть пути работает: метрики есть, но dashboard декоративный. Или Sentry ловит ошибки, но Loki не настроен. Интервьюер может частично воспроизвести сигналы. |
| 17–21 | 4 типа сценариев работают. Метрики в Prometheus, логи в Loki, ошибки в Sentry. Dashboard с 3+ панелями. Verification walkthrough проходится за 5 минут. |
| 22–25 | Всё из 17–21, плюс: метрики осмысленно названы, логи фильтруемы по scenarioType, dashboard реально полезен (не "hello world" панели), Loki panel в Grafana. |

---

## 3. Cursor AI Layer — 25 баллов

| Баллы | Что видит интервьюер |
|-------|---------------------|
| 0–8 | Файлы есть, но формальные: rules пустые или копипаста, skills без конкретных инструкций, commands не работают. |
| 9–16 | Skills и rules полезны, но не все scoped. Marketplace skills подключены, но выбор не объяснён. Hooks декларативные. |
| 17–21 | Каждый артефакт имеет чёткую роль. Custom skills с хорошими "When to Use". Commands ускоряют реальные workflows. Rules предотвращают типовые ошибки. Marketplace skills обоснованы. |
| 22–25 | Всё из 17–21, плюс: новый чат Cursor реально получает контекст, hooks ловят настоящие проблемы, документация AI-слоя лаконичная но полная. Видна система, а не набор файлов. |

---

## 4. Orchestrator — 15 баллов

| Баллы | Что видит интервьюер |
|-------|---------------------|
| 0–5 | Есть prompt, но нет декомпозиции, нет context file, нет model selection. По сути большой единый промпт. |
| 6–10 | Есть context.json и фазы. Задачи разбиты, но model selection формальный ("всё fast"). Retry/resume не работает. |
| 11–15 | Atomic decomposition убедительна. Задачи помечены fast/default с обоснованием. Context file позволяет resume. Orchestrator связан с другими skills. Финальный отчёт читаем. |

---

## 5. Документация и DX — 10 баллов

| Баллы | Что видит интервьюер |
|-------|---------------------|
| 0–3 | README есть, но запуск требует угадывания. Нет verification walkthrough. |
| 4–7 | README достаточен для запуска. Есть инструкция проверки. AI-слой описан списком. |
| 8–10 | README позволяет проверить всё за 15 минут. AI-слой задокументирован с "зачем" а не только "что". Скриншоты или видео — бонус. |

---

## Штрафы

Штрафы применяются **после** подсчёта баллов. Минимальный итоговый балл: **0**.

| Штраф | Когда |
|-------|-------|
| **−15** | Обязательный стек заменён без аргументации |
| **−15** | Нет рабочего verification walkthrough для observability |
| **−10** | Нет ни одного custom Cursor skill |
| **−10** | Нет marketplace skills или они не объяснены |
| **−10** | Нет hooks или commands |
| **−5** | Решение нельзя поднять по README |

Максимум штрафов: **−50** (но сумма баллов не может быть меньше 0).

---

## Бонус

В PRD спрятан необязательный бонусный сценарий. Если нашёл и реализовал — **+5 баллов** сверх 100.

---

## Итоговая шкала

| Балл | Рекомендация |
|------|-------------|
| 80–100 | **Strong hire** — системное мышление, рабочий результат, AI-слой как продукт |
| 65–79 | **Hire** — хорошая база, есть пробелы, но фундамент крепкий |
| 50–64 | **Lean hire** — код работает, но AI-слой слабый или observability формальная |
| 30–49 | **Lean no** — значительные пробелы, но видно понимание направления |
| 0–29 | **No hire** — не выполнены базовые требования |
